マイクロソフト社の最新サーバ向けオペレーティングシステムであるWindows Server 2008がリリースされてから2年、その機能追加版であるWindows Server 2008 R2のリリースからも半年以上が経過しました。ある程度実績が出てから自社でも導入を検討しようと様子を見られている企業様も、そろそろ実導入を検討する段階に入ってきているのではないでしょうか。

そこで、今回の記事では、Windows Server 2008で新しく追加された機能から、主だったものを取り上げて、Windows Server 2008 の導入メリットを3点ご紹介します。

Server Core - 新機能1

特定の枠割・機能のための最少環境を構成

Server Coreは、Windows Server 2008 で追加された新しいインストールオプションです。DNSやDHCPなど、特定の役割・機能を利用するために必要となる最小の環境を構成します。


このオプションを選択してOSをインストールすると、サーバには必要最低限のコンポーネントのみが導入されます。エクスプローラをはじめとするGUIもインストールされないため、コマンドでの管理が基本になります。

リモートからGUI管理も可能

しかしこのように記載すると、これまでのWindowsに慣れた管理者の方は思わず身構えてしまうかもしれません。しかしServer Coreオプションでインストールされたサーバも、リモートのGUIから接続して操作することができます。


Windows Vista、Windows 7用には、RSAT（Remote Server Administration Tools）というリモート管理ツールが無償で提供されているため、たとえば管理者のPCにRSATをインストールし、RSATのイベントビューアから、Server Coreオプションでインストールしたサーバのイベントを参照することも可能です。

Server Core オプションを利用する利点

  • 必要最低限のサービスしか実行されないため、悪意あるユーザーからの攻撃対象となり得る部分が狭く、セキュリティ性が高い
  • インストールに必要となるハードウェアスペックが、フルインストールよりも低い

※ 要件はすべて推奨要件です
インストール オプション メモリ要件 ハードディスク要件
フルインストール 4GB以上 40GB以上の空き領域
Server Core インストール 2GB以上 10GB以上の空き領域

なお、Server Core は、アプリケーションプラットフォームではないため、基本的に、Exchange Server や SQL Server などのアプリケーションをインストールすることはできません。Server Core は、あくまでもWindowsの役割と機能をシンプルな構成で利用するためのオプションです。

Server Core で利用できる役割・機能

[役割]
AD DS, AD LDS, DHCP, DNS, ファイルサービス, プリントサーバ, ストリーミングメディアサービス, Webサーバ(IIS), Hyper-Vなど


[機能]
MSFC, NLB, Windowsバックアップ, BitLocker, SNMP, WINSなど

Hyper-V - 新機能2

仮想化で複数のサーバーを集約

Hyper-Vは、ハイパーバイザー型の仮想化アプリケーションで、この機能を利用すれば、1台の物理サーバ上で複数のOSを起動することが可能です。サーバ仮想化は、サーバ統合の観点からも注目を浴びている機能の1つで、異なるサービスを提供する複数のサーバを、1台の物理サーバに集約することで、運用管理コストを抑える効果が期待できます。

仮想インスタンスのライセンス

また、Windows Server 2008 Enterprise Edition では、1ライセンスにつき4台まで、Windows Server 2008 Datacenter Edition では、1ライセンスにつき無制限の仮想インスタンスを実行することができるため、ライセンスコスト面でもメリットがあります。


既存の物理サーバを仮想化したいといった場合には、System Center Virtual Machine Manager 2007の物理サーバ変換ウィザードを利用したP2Vも可能です。

Hyper-VでサポートされるOS

  • Windows Server 2008
  • Windows Server 2003
  • Windows 2000 Server SP4以上
  • Windows 7
  • Windows Vista SP1以上
  • Windows XP SP2以上
  • SUSE Linux Enterprise Server 10 SP1以上

ネットワークアクセス保護（NAP） - 新機能3

NAPはポリシーベースのセキュリティプラットフォーム

NAP（Network Access Protection）は社内リソースの保護を強化するためのポリシーベースのセキュリティプラットフォームです。ネットワークに接続したクライアントの状態を自動的にチェックし、そのクライアントがあらかじめ定められたセキュリティ基準を満たしていない場合には、社内リソースへの接続を制限することで、社内ネットワークを保護する仕組みです。

NAPの仕組み

NAPが保護するネットワークに接続したクライアントは、「検疫ゾーン」と呼ばれる制限されたネットワークに所属し、セキュリティパッチやウィルス対策ソフトなどのセキュリティ状態のチェックを受けます。その結果、ポリシーを満たしている場合は、「セキュアゾーン」に移動して社内リソースにアクセスすることができます。


ポリシーを満たしていないクライアントに対しては、WSUSサーバからセキュリティパッチを提供するなどして足りない要件を満たしてから、社内リソースへのアクセスを許可します。


この機能を利用することで、「コンピュータ導入時に設定したはずのセキュアな状態が維持できない」というシステム担当者様の問題点をクライアント端末のセキュリティ管理を自動化することにより解決します。


なお、NAPは、社内のポリシーに準拠しない（セキュリティ性の低い）PCの接続を制限する機能であって、悪意あるユーザーのアクセスを制限するための機能ではありません。不正アクセスの防止には、別のシナリオを検討してください。
