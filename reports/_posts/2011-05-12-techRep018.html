世の中の企業のほとんどは多種多様なシステムを導入され、業務の効率化という観点において大きな進展がある企業が多いのではないでしょうか？

しかし、逆に多くのシステムを導入されているという事はシステムごとに必要な情報のライフサイクル管理（ユーザID/パスワード情報）が煩雑になっているのも事実だと思います。


今回ご紹介するMicrosoft Identity Lifecycle Manager 2007とは上記のような問題点を解決する為の強力なツールとなります。

利用者の現状

利用者の観点として、パソコンを起動すると最初に、WindowsのユーザID/パスワードの入力を求められます。

その後、システムによっては業務システムを起動する際や、ポータルサイト、経費等の精算システムへのアクセスする際にもユーザID/パスワードを入力するケースが発生し、利用するシステムごとに簡易的なパスワードを使用したり、パスワードをメモ書きすると言ったセキュリティ上問題がある運用をされている企業も多く存在していたのではないでしょうか？


また上記に加えて現在では、セキュリティポリシーによっては3カ月に1度はパスワードの変更をする必要も出てきております。

管理者の現状

管理者の観点としては、大きく分けて二つのコストがかかっていたかと思います。

  • 利用者の情報同期及び初期化：利用者が利用するシステムのID情報の同期や、利用者が運用中にパスワードを紛失した場合などの利用者の対応。
  • ユーザのメンテナンス：新しい社員が入ってきた場合や退職者が出た場合など、ユーザーアカウントの追加・削除等の作業。

そういった問題点を解決する為にMicrosoft Identity Lifecycle Manager 2007が存在します。

ILM2007を導入する際のメリット

上記のような状況をILM2007を導入する事によりこれまでかかっていた運用コストを削減する事だけでなく、情報に関するセキュリティをアップさせる事が可能になります。

ID情報の同期

全てのシステムのユーザアカウントと属性を同期する事が可能になります。この事により、従来それぞれに管理していたアカウント情報管理をせずにデータの一貫性を保ち、従来かかっていた運用コストを大幅に削減する事が可能となります。

ユーザアカウントの準備

全てのシステムのユーザアカウント情報や属性がリアルタイムに作成されます。


よって、これまで社員のメンテナンス（新しい社員や退職者が出た場合など）のたびに担当者が抱えていた作業を簡易的に実施する事が出来ます。

この事により、これまでのヘルプデスクにかかっていた費用を削減する事も可能になります。

証明書とスマートカードの管理

旧バージョンのMIIS2003では実現できなかったデジタル証明書やスマートカードの管理が可能になりました。

よって証明書ベースの展開・管理・維持等が出来るようになるため、運用コストを削減する事が可能になります。

ILMイメージ構成

ILMを導入すると以下の図のように、それぞれのシステムで必要なID情報をILMで集約しID同期を行うイメージになります(パスワード情報含む)。よって、各アプリケーションへ個別にアカウントの設定等を実施する必要がなくなります。

ILMシステム要件

ILMを導入する為に必要なサーバー環境は以下の通りです。

ILM導入

それではMicrosoft Identity Lifecycle Manager 2007を導入する際に必要となる内容について考えていきます。

1．社内で利用されているアカウント情報やID情報を整理する

まず現在社内で利用されているアプリケーションでIDレポジトリがどこに格納されているのか調査を実施する必要があります。


例)

  • Active Directory
  • 業務アプリケーションA(SQL Server 2005)
  • 業務アプリケーションB(Oracle9i)
  • 人事システム
  • 電子メール等々
    ※ID情報だけでなく、その他必要になる項目(パスワード含む)を洗い出します。

2．設計・構築

上記で整理されたアカウント概要に基づき設計

各アプリケーションのレポジトリに基づき概要・実装方針を検討します。
パスワードの管理方針を検討します。

環境について検討

  • ネットワーク構成
  • ILMサーバの設計・構築
  • ILM環境の構成（メターバース、コネクタスペース）

3．運用方針検討

今後のアカウント運用方針を検討する必要があります。
ILMサーバのバックアップリストア手法を検討する必要があります。

ILM導入後

ILMを導入すると、これまで運用でカバーリングされていた内容が自動化されるため、運用コストの削減及びセキュリティの向上が可能となります。またSSO（シングルサインオン）を実現も可能となります。他にも、ワークフロー管理機能もあるため、IT管理者はこの機能を利用する事も可能です。


通常Active Directoryのパスワードを変更した場合、ILMによって他のアプリケーションのパスワードも同期されますので、複数のパスワードを別々に管理する必要はありません。


もし企業内に様々なアプリケーションが存在し、運用に高負荷がかかっているようであればILMの検討をしてみてはいかがでしょうか？
