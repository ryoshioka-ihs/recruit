「仮想化」は、現在のIT業界で最も注目を集めているキーワードの1つです。この数年の間に新しい製品が次々とリリースされ、同時に機能強化が図られてきました。


今回は、数ある仮想化製品の中から特に取り上げられることの多い「Hyper-V」と「VMWare ESX」を取り上げて、両者の違いをご紹介していきたいと思います。


それでは実際の比較に入る前に、まずそれぞれの製品について簡単に説明します。

Hyper-V

マイクロソフトが提供するハイパーバイザー型の仮想化ソリューションです。

Windows Server 2008に標準で組み込まれており、同社のSystem Center製品群と連携することで仮想環境の統合管理を実現します。

VMWare ESX

ヴィエムウェアが提供するハイパーバイザー型の仮想化ソリューションです。

VMWare vSphere（VMWare Infrastructure 3の新バージョン）のコンポーネントの1つで、他のvSphereコンポーネントと連携することで仮想環境の統合管理を実現します。

なお、今回の記事では、特に断らない限り、次のバージョンを前提として記述します。

  • Hyper-V 2.0 ※
  • VMWare ESX 4.0

※近日リリース予定のWindows Server 2008 R2に搭載予定の新バージョン。リリース前に発表された内容を元にしているため、実際に発売された製品の仕様は異なっている可能性があります。

アーキテクチャの違い

Hyper-VとVMWare ESXは、ともにハイパーバイザー型（ホストOSを必要としない）の仮想化ソリューションです。ただし、Hyper-Vは「ペアレントOS」と呼ばれる管理用のOSを必要とし、ゲストOSはVMBusを経由してペアレントOSのデバイスドライバに接続します。


一方、VMWare ESXでは、デバイスドライバはハイパーバイザー層に包含されています。


このため、パフォーマンス面ではペアレントOSを介さないVMWare ESXが優れ、ハードウェア互換性の面ではハイパーバイザー層の外にデバイスドライバを持つHyper-Vに分があると言えます。

機能の違い

Hyper-VとVMWareESXは、主としてハイパーバイザー機能を提供する製品です。どちらも単独で使用できますが、連携する他の製品と組み合わせることにより、より高度な機能を利用できるようになります。


Hyper-V 1.0とVMWare Infrastructure 3の時点では、Live Migration（仮想マシンをオンラインの状態のままで別の物理サーバに移動させる機能）の有無など、機能面においてVMWare側に1日の長がありました。


しかしながら、Hyper-V 2.0での機能追加・強化を受け、単純な機能の有無という点では、ほぼ両者に差は見られなくなっています。 両者が提供する主な機能と、その機能を利用するために必要な連携製品を、次にまとめます。

機能面での比較とは一転して、サポートするOS面ではVMWare ESXが群を抜いています。 Hyper-VでもLinuxに対応するなどして範囲を広げる動きが見られますが、Windows NTのようなレガシクライアントについては対応の可能性は低いでしょう。


一方、ヴイエムウェアはvSphere製品群を「データセンターのOS」として位置付けており、この面での優位性は揺るぎないものと言えます。

導入コスト

最後に、導入コストについて考えてみましょう。まず、ライセンス費用の問題ですが、ゲストOSがWindowsの場合には、一般的にHyper-Vのほうがコストを低く抑えることができます。


Windows Server 2008のStandard Edition、Enterprise Edition、Datacenter Editionには、ホスト1台に対してエディションに応じた仮想インスタンスライセンスが付属しています。たとえば、Enterprise Editionのライセンスでは、ホスト1台とゲスト4台の計5台を使用することができます。


その一方で、パフォーマンス面ではVMWare ESXが優れており、1台のホスト上で稼働させられる仮想サーバ数に差が出ることになります。すなわち、一定以上の規模の環境では、物理サーバ台数を抑えられるVMWare ESXのほうが安価に抑えられる可能性もあります。

まとめ

以上の点をまとめると、現時点では両製品の位置づけを次のように考えることができます。

Hyper-V

  • Windows基盤の小～中規模環境、テスト・検証環境を構築する場合に有利です。
  • Windows系のスキルを持つ管理者・運用者が、低コストで仮想環境を構築したい場合に適しています。

VMWare ESX

  • 異なるOS・バージョンのシステムを含む大規模環境を構築する場合に有利です。
  • 多種多様な環境を一元的に運用・管理したい場合に適しています。
