System Center Configuration Manager 2007（SCCM 2007）を利用したIT運用管理の効率化の中からOS展開機能について、詳しくご紹介したいと思います。

昨年秋、マイクロソフトからWindows 7がリリースされました。この機会にコンピュータの新規導入やリプレイス、バージョンアップを検討されている企業様も多いのではないでしょうか。SCCM 2007では、システム部門にとって最も負荷の高い作業の１つであるPCのセットアップ作業の大部分を自動化することが可能です。

SCCM 2007で展開可能なOS

SCCM 2007で展開可能なOS

  • Windows 2000 Professional SP4以上
  • Windows XP Professional SP2以上
  • Windows Vista
  • Windows 7
  • Windows 2000 Server SP4以上
  • Windows Server 2003 SP1以上
  • Windows Server 2008

SCCM 2007によるOS展開の流れ

SCCM 2007のOS展開プロセスは、次の通りです。

  • 雛形コンピュータをセットアップする

    実際に対象コンピュータへ展開するのと同じ構成で、OS、ドライバ、アプリケーション、更新プログラムなどをインストールした雛形コンピュータを準備します。

  • OSイメージをキャプチャする

    SCCMでキャプチャメディア（OSイメージをキャプチャするためのプログラム）を作成し、雛形コンピュータで実行すると、雛形コンピュータの構成をWIMファイル（※注1）として取得できます。このWIMファイルをもとに、SCCMでOS展開用パッケージを作成します。

  • セットアップ実行ステップ（タスクシーケンス）を定義する

    ディスクのフォーマット→OSセットアップ→更新プログラム適用→ドメイン参加 … といった一連のセットアップ実行プロセスを定義します。この作業は、簡単なウィザード形式で設定することができます。

  • 対象コンピュータにOSを展開する

    2.で作成したOSイメージのパッケージと、3.で定義したタスクシーケンスに基づいて、対象コンピュータにOSを展開します。


※注1…WIMファイル：
WIMはWindows Vistaで導入されたファイルベースのディスクイメージ形式です。WIMファイルは、多数の関連ファイルを格納した圧縮パッケージです。

4種類のOS展開方式

SCCM 2007では、4種類のOS展開方法が提供されています。

IT管理者は、実際の環境やニーズに合わせて、適当な方法を選択することができます。

(1) セットアップメディアを使用する


OSイメージ、タスクシーケンスを含めたセットアップメディアを作成し、それを利用して対象コンピュータをセットアップします。対象コンピュータとSCCMサーバの間でデータのやりとりが発生しないため、低速なネットワークで接続された環境でのOS展開に適しています。

(2) ブートメディア＊を使用する


対象コンピュータをブートメディアで起動し、SCCMに接続してOSイメージ、タスクシーケンスを取得して対象コンピュータをセットアップします。

(3) PXEブート＊を使用する


Windows展開サービス（WDS）と連携し、対象コンピュータをPXEブートしてSCCMに接続し、OSイメージ、タスクシーケンスを取得して対象コンピュータをセットアップします。 メディアを使用することなくOSを新規インストールできます。

(4) SCCMエージェントを使用する


SCCMエージェントからSCCMサーバに接続し、OSイメージ、タスクシーケンスを取得して対象コンピュータをセットアップします。Wake On LAN（※注2）にも対応しているため、夜間の無人アップグレードなど、スケジュールに基づいたゼロタッチのOS展開も可能です。


既にOSがインストールされ、SCCMの管理下にあるコンピュータにOSを展開（アップグレード）する場合にのみ使用できます。全くの新規インストールには使用できません。


このように、SCCM 2007では、様々な状況や環境に応じた柔軟なOS展開を実現することができます。また、OS展開作業を自動化することは、コンピュータ構成の標準化にも繋がるため、展開後の運用管理を効率的に進めることができるようになります。


・※注2…Wake On LAN：
LANに接続されたコンピュータをネットワーク経由で他のコンピューター経由で他のコンピュータから起動する機能。
